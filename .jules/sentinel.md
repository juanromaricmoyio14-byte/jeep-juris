## 2026-05-30 - Sanitize error logs and CSP Implementation
**Vulnerability:** Verbose API error texts and full stack traces were logged into standard console error outputs on the server. Additionally, raw CSS generated from dynamic configuration colors was injected into `dangerouslySetInnerHTML` without sanitization.
**Learning:** While `dangerouslySetInnerHTML` in `<style>` contexts limits immediate script injection without breaking the tag (e.g. `</style><script>`), failing securely means eliminating even theoretical paths by sanitizing dynamic payload construction. Exposing direct stack traces or external API responses (e.g., Gemini) could expose internal architecture or model hints.
**Prevention:** Wrap inline `dangerouslySetInnerHTML` injections with a library like `isomorphic-dompurify`, explicitly define safe CSP headers, and normalize error logs using descriptive, static string templates over passing dynamic `Error.message` content from dependencies directly into logs.
